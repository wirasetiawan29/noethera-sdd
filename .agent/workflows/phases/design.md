# Phase 2: Design

Before Implement. Do not code until design is explicit.

## Checklist

- [ ] **Layers** — Handler → Service → Repository, or equivalent. Clear boundaries.
- [ ] **Interfaces** — Public contracts; dependencies injectable.
- [ ] **Patterns** — Which pattern(s)? No anti-patterns (see architecture-and-patterns).
- [ ] **Impact** — For updates: what files/modules change? Side effects?

## Reference rules

- `core-design-principles` — SOLID, DRY, KISS, YAGNI
- `architecture-and-patterns` — Layering, anti-patterns
- `avoid-circular-dependencies` — No A↔B cycles
